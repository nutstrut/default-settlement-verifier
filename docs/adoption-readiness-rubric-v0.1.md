# Adoption Readiness Rubric v0.1

## Purpose

This document defines the evaluation criteria used to assess adoption readiness across all Default Settlement public surfaces.

The rubric is used for:

- Website audits
- Documentation audits
- GitHub audits
- Directory listing audits
- Onboarding reviews
- Morpheus adoption-readiness monitoring

The objective is to identify friction, messaging drift, routing gaps, and onboarding bottlenecks before amplification efforts increase traffic.

---

# Adoption Readiness Evaluation

Each surface should be evaluated against the following criteria.

---

## 1. Comprehension

### Question

Can a visitor answer:

"What is this?"

within 60 seconds?

### Pass Criteria

A first-time visitor can accurately describe the purpose of the surface without prior knowledge of SAR, Default Settlement, or agent governance concepts.

### Failure Indicators

- Excessive jargon
- Undefined acronyms
- Technical terminology before problem explanation
- Ambiguous purpose

---

## 2. Relevance

### Question

Can a visitor answer:

"Why should I care?"

within 60 seconds?

### Pass Criteria

The surface clearly communicates the problem being solved and why it matters to the intended audience.

### Failure Indicators

- Features presented before value
- Technical implementation emphasized over outcomes
- No clear problem statement

---

## 3. Audience Routing

### Question

Can a visitor determine:

"Which path applies to me?"

within 60 seconds?

### Primary Audiences

- AI Users
- Developers
- Infrastructure Builders
- Enterprise / Governance Users

### Pass Criteria

The visitor can identify their audience category and find an appropriate next step.

### Failure Indicators

- One-size-fits-all messaging
- No audience segmentation
- No audience-specific guidance

---

## 4. Next-Step Clarity

### Question

Can a visitor answer:

"What should I do next?"

within 60 seconds?

### Pass Criteria

The surface provides a clear action path.

Examples:

- Generate a receipt
- Verify a receipt
- Read documentation
- Explore the architecture
- View the explorer

### Failure Indicators

- Dead-end pages
- No call-to-action
- Multiple competing actions without guidance

---

## 5. Narrative Alignment

### Question

Does the surface reinforce the canonical narrative?

### Canonical Narrative Themes

- Independently verifiable execution outcomes
- Execution boundary infrastructure
- An AI agent says: Task Complete. How do you know?
- If it matters — verify it.

### Pass Criteria

The surface reinforces the same story and positioning used throughout the ecosystem.

### Failure Indicators

- Conflicting positioning
- Legacy messaging
- Inconsistent terminology

---

## 6. Credibility Signals

### Question

Does the surface provide evidence that the system is real and operational?

### Examples

- Live receipts
- Public metrics
- Public verification keys
- Running infrastructure
- Open-source implementations
- Ecosystem participation

### Pass Criteria

A visitor can observe objective evidence that the system is functioning.

### Failure Indicators

- Pure marketing claims
- No operational proof
- No examples or demonstrations

---

## Scoring

Each category should be scored:

- 1–3 = Weak
- 4–6 = Needs Improvement
- 7–8 = Strong
- 9–10 = Excellent

---

## Audit Output Format

Each audit should include:

### Surface

Name of surface being evaluated

### Audience

Primary audience(s)

### Scores

- Comprehension
- Relevance
- Audience Routing
- Next-Step Clarity
- Narrative Alignment
- Credibility Signals

### Findings

Key observations

### Friction Points

Specific adoption barriers

### Recommendations

Prioritized improvements

### Result

PASS / FAIL

---

## Objective

The goal is not to maximize traffic.

The goal is to maximize successful understanding, onboarding, and adoption once traffic arrives.

Adoption readiness should be validated before amplification efforts increase visibility.

If it matters — verify it.
