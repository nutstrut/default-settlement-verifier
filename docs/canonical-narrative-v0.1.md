## Problem Statement

AI agents are moving from conversation to execution.

They send payments, invoke tools, modify systems, and perform work on behalf of people and organizations.

As agent capabilities grow, a new question emerges:

**An AI agent says: Task Complete. How do you know?**

Claims, confidence scores, and transcripts are useful, but they do not provide independently verifiable evidence that a task satisfied its specification.

As agents become more capable and autonomous, independently verifiable execution outcomes become increasingly important.

---

## Category Definition

Default Settlement provides independently verifiable execution outcomes for agent workflows.

Through SAR (Settlement Attestation Receipts), agent actions can be evaluated against their specifications and recorded as cryptographically signed receipts that any party can verify independently.

SAR receipts are portable, machine-readable, and cryptographically verifiable.

They can be used across payment, settlement, governance, compliance, reputation, and agent-to-agent workflows.

## What Default Settlement Does

Default Settlement focuses on execution.

Other systems may describe:

- Identity
- Intent
- Reasoning
- Governance
- Delegation
- Continuity

Default Settlement focuses on what happened at execution and whether the outcome can be independently verified.

This creates an execution boundary between claims and proof.

## Messaging Principles

### Lead with the problem

Start with the question:

**"How do you know?"**

Avoid leading with technical terminology.

---

### Explain value before implementation

Visitors should understand why independently verifiable execution outcomes matter before learning about SAR receipts, signatures, or APIs.

---

### Emphasize independent verification

The key property is not trust.

The key property is independent verification.

---

### Position SAR as execution-boundary infrastructure

Default Settlement occupies the execution layer of the emerging agent stack.

The focus is execution outcomes and verifiable proof.

---

### Reinforce the same narrative across all surfaces

Every public surface should reinforce:

- Independently verifiable execution outcomes
- Execution boundary infrastructure
- An AI agent says: Task Complete. How do you know?
